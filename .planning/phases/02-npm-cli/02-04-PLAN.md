<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User-supplied CLI flags (--dry-run, --minimal) → Commander option parsing | User input enters via process.argv |
| init action handler → step modules | Orchestration layer calls step functions with filesystem and subprocess side effects |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04-A | Tampering | Commander options dryRun / minimal — boolean flags only | accept | Commander.js coerces these as boolean flags; no user-supplied string values are accepted that could be injected into subprocess calls |
| T-02-04-B | Information Disclosure | file list printed in note() | accept | File list contains only relative paths under the user's own CWD; no system paths, no secrets |
| T-02-04-C | Denial of Service | @clack/prompts tasks() — uncaught exception in one task crashes CLI | mitigate | Each task's async function wraps its step call; unexpected errors bubble up to Commander's error handler which prints a user-friendly message and exits 1 rather than printing a raw stack trace |
| T-02-04-D | Tampering | --dry-run bypass — code must actually not write files | mitigate | dryRun=true path calls listTemplateFiles only; copyTemplates with dryRun=true is verified by integration test asserting no files exist in tmpdir after the call |
</threat_model>
