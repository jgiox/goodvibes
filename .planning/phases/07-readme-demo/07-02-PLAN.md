    1. From the repo root: `vhs scripts/demo.tape`

    3. Verify size < 2MB: `test $(wc -c < docs/demo.gif) -lt 2097152 && echo "size OK"`
    4. Open docs/demo.gif in an image viewer and confirm:
       - Dark Dracula theme visible
       - Terminal width looks approximately 800px
       - `npx goodvibes init --minimal --dry-run` command is shown being typed
       - Completion output shows written/skipped file counts (Phase 6 hardened output)
       - No npm download noise visible before the goodvibes output

    **Option B — VHS not installed locally:**
    1. Commit scripts/demo.tape (Task 1 output) to main
    2. Wait for Plan 3 (vhs.yml) to be created and triggered
    3. CI will produce and commit docs/demo.gif automatically
    4. After CI run: `git pull && ls -lh docs/demo.gif`
    5. Verify size: `test $(wc -c < docs/demo.gif) -lt 2097152 && echo "size OK"`

    **If GIF exceeds 2MB (Pitfall 5 mitigation):**
    - Edit scripts/demo.tape: add `Set PlaybackSpeed 1.5` after `Set Framerate 24`
    - Re-run vhs or push updated tape; repeat size check

    **If `Wait+Screen /\$/ 60s` times out in CI:**
    - The VHS step will fail with a timeout error in the CI log
    - Increase timeout: change `Wait+Screen /\$/ 60s` to `Wait+Screen /\$/ 90s` in the tape

  </how-to-verify>
