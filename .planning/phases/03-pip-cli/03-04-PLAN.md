<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI → copy_templates/install_headroom/configure_mcp | Internal calls to tested modules; no user input flows into subprocess args |
| GitHub Actions → PyPI | Trusted publishing via OIDC; no API token stored in repo |
| cwd (process.cwd() equivalent) | User's working directory is the target for file writes |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04-01 | Tampering | cwd used as dest_dir | accept | User intentionally runs goodvibes init in their project directory; no path validation needed at CLI level (copy_templates has path traversal guard) |
| T-03-04-02 | Repudiation | PyPI publish without trusted publishing | mitigate | publish-pip.yml uses --trusted-publisher always; no token in repo; OIDC attestation on PyPI |
| T-03-04-03 | Denial of Service | headroom install hangs CLI | mitigate | subprocess.run has no timeout per se, but install_headroom soft-fails on CalledProcessError; ONNX warning primes user expectation |
| T-03-04-04 | Information Disclosure | version guard stderr message | accept | Error reveals Python version in use; acceptable for a developer tool |
| T-03-04-SC | Tampering | npm/pip/cargo installs | mitigate | No new runtime deps added in this plan; publish workflow uses uv (system tool) |
</threat_model>
