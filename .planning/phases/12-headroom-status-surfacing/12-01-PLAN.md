<objective>
Refactor the two TypeScript step functions to return discriminated union result types instead of void, change the headroom idempotency probe from `--version` to the functional `compress --help`, and add a 10-second timeout to every execa call in both step files.

Purpose: HDR2-03 and HDR2-04 address the root cause of two bugs — broken headroom installs that pass the `--version` probe (Windows + Python 3.13), and subprocesses that hang indefinitely. This plan changes the steps layer only; the commands layer (init.ts, doctor.ts) is updated in Plan 12-02.

Output: `HeadroomResult` and `McpResult` exported types; both step functions return structured results; all tests GREEN.

TDD discipline: Task 1 updates tests to the RED state and commits. Task 2 updates implementations to the GREEN state and commits.
</objective>
