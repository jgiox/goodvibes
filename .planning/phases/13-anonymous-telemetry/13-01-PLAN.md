    Create the directory workers/telemetry/ and write three files.

    worker.js: A plain JavaScript (not TypeScript — no build step needed per RESEARCH.md Don't Hand-Roll) Cloudflare Worker export default object with a single fetch() handler. The handler rejects non-POST by returning new Response(null, { status: 204 }) immediately (per D-02 and D-06). For POST requests: read env.INSTALLS.get('total') (defaults to '0' if null), increment, write back with env.INSTALLS.put('total', String(n+1)). Then read env.INSTALLS.get(today) where today = new Date().toISOString().slice(0, 10) (YYYY-MM-DD format per D-02), increment, write back. Return new Response(null, { status: 200 }). The handler reads nothing from the request object beyond request.method — no IP (req.headers.get('cf-connecting-ip')), no body, no User-Agent. Mark this constraint with a ponytail: comment.

    wrangler.toml: Set name = "goodvibes-telemetry", main = "worker.js", compatibility_date = "2025-01-01", workers_dev = true. Add [[kv_namespaces]] section with binding = "INSTALLS" and id = "PLACEHOLDER_KV_NAMESPACE_ID". The placeholder ID will be replaced in Plan 13-06 after wrangler kv:namespace create INSTALLS is run. Add a comment above the id line: "# replace with output of: wrangler kv:namespace create INSTALLS".

    package.json: Minimal JSON with name "goodvibes-telemetry", private true, and devDependencies containing wrangler at "^4". No scripts needed — wrangler commands are run via npx or the wrangler-action in CI.

    Do NOT add workers/telemetry/ to the root package.json or any monorepo workspace config — it is standalone.
