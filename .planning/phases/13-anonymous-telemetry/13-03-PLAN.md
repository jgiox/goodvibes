    Public start_telemetry_thread() -> threading.Thread | None function: if _opt_out() returns None. Otherwise generate request_id = str(_uuid.uuid4()) (per TEL-02 — per-invocation, never stored), create t = threading.Thread(target=_fire, args=(request_id,), daemon=True), call t.start(), return t.

    The daemon=True ensures Python kills the thread automatically if the process exits before join() is called (Pitfall 5 in RESEARCH.md — safe by design).

    No __all__ needed — the module exports by convention. The type annotation threading.Thread | None requires Python 3.10+ which the project already requires.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI process (Python) → TELEMETRY_URL | Outbound POST from user machine; network untrusted |
| os.environ → _opt_out() | Env vars read at call time; must be checked before thread is started |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-03-DRY | Spoofing | start_telemetry_thread() call site | mitigate | start_telemetry_thread() has no dry-run check itself; dry-run guard enforced in init_cmd.py (Plan 13-05) — call only inside non-dry-run path |
| T-13-03-UUID | Information Disclosure | X-Request-Id header | accept | UUID is per-invocation, non-persistent, non-identifiable; not linked to user; Worker does not log or store headers (D-02) |
| T-13-03-HANG | Denial of Service | urlopen blocking indefinitely | mitigate | urlopen(req, timeout=5) caps individual HTTP call; thread.join(timeout=1.0) in init_cmd.py (Plan 13-05) caps user-visible wait |
| T-13-03-THREAD | Information Disclosure | daemon thread leaking on exception | accept | daemon=True; Python auto-kills daemon threads on process exit; no PII in thread, no data-loss risk (RESEARCH.md Pitfall 5) |
</threat_model>
