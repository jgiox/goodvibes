<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Human → GitHub Secrets | API token must transit only through the GitHub Secrets UI — never into source code |
| wrangler.toml → Cloudflare CI | KV namespace ID is a non-secret config value (public in repo); API token is always a secret |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-06-CRED | Information Disclosure | CLOUDFLARE_API_TOKEN handling | mitigate | Token stored only in GitHub Secrets; never appears in wrangler.toml, worker.js, or any committed file; wrangler-action reads it as ${{ secrets.CLOUDFLARE_API_TOKEN }} |
| T-13-06-KV_ID | Information Disclosure | KV namespace ID in wrangler.toml | accept | KV namespace ID is a non-credential config value (analogous to a database name, not a password); public in repo is acceptable — Cloudflare requires API token auth to read/write KV |
| T-13-06-URL | Tampering | GOODVIBES_TELEMETRY_URL env override on published package users | accept | Env override is opt-in for contributors; no end user sets this; even if overridden, result is a different URL receiving the POST — no PII risk since counter stores no identifiable data |
</threat_model>
