| Boundary | Description |
|----------|-------------|
| CLI process → TELEMETRY_URL | Outbound HTTP POST from the user's machine to the Worker; network untrusted |
| process.env → sendTelemetry() | Opt-out env vars read from the environment; must be checked before any network call |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-02-DRY | Spoofing | sendTelemetry() call site | mitigate | sendTelemetry() has no dry-run check itself; dry-run guard is enforced in init.ts (Plan 13-04) — sendTelemetry() must only be called inside the non-dry-run code path |
| T-13-02-UUID | Information Disclosure | X-Request-Id header | accept | UUID is per-invocation, cryptographically random, never stored to disk (TEL-02); not linkable to user identity; Worker does not log it |
| T-13-02-HANG | Denial of Service | fetch() blocking indefinitely | mitigate | AbortController aborts after 5 seconds; Promise.race in init.ts adds 1-second cap from user perspective (TEL-05) |
| T-13-02-SC | Tampering | npm install of wrangler (workers/telemetry only) | accept | No new packages in packages/npm — telemetry uses Node 20 built-in fetch; zero new npm deps in CLI package |
</threat_model>
