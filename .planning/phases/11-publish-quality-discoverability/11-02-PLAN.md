    Append a new job named `check-stamps` to .github/workflows/ci.yml. Add it after the existing `smoke-test` job. The job must:

    1. Run on ubuntu-latest with no `needs` dependency (runs in parallel with existing jobs).
    2. Use `actions/checkout@v7`.
    3. Have a single step named "Check version stamps are in sync" with this exact shell script:

    NPM_VER=$(grep '"version"' packages/npm/package.json | head -1 | sed 's/.*"version": "\(.*\)".*/\1/')
    PIP_VER=$(grep '^version' packages/pip/pyproject.toml | sed 's/version = "\(.*\)"/\1/')
    TPL_VER=$(grep '^# goodvibes: v' templates/CLAUDE.md | sed 's/# goodvibes: v//')

    echo "npm:      $NPM_VER"
    echo "pip:      $PIP_VER"
    echo "template: $TPL_VER"

    if [ "$NPM_VER" != "$PIP_VER" ] || [ "$NPM_VER" != "$TPL_VER" ]; then
      echo ""
      echo "ERROR: Version stamps are out of sync."
      echo "Fix: update templates/CLAUDE.md header to '# goodvibes: v$NPM_VER'"
      echo "Fix: update packages/pip/pyproject.toml version to match packages/npm/package.json"
      exit 1
    fi
    echo "All stamps in sync: v$NPM_VER"

    All three version extractions use grep + sed (POSIX shell, no node required). The sed patterns use basic regex (no -E flag) for GNU + BSD sed compatibility.

    Do not change any existing jobs (test-npm, test-pip, smoke-test).

  <action>
    In .github/workflows/publish-npm.yml, append a new job `smoke-test` after the existing `publish` job:

    smoke-test:
      needs: publish
      runs-on: ubuntu-latest
      steps:
        - name: Wait for registry propagation
          run: sleep 30

        - uses: actions/setup-node@v6
          with:
            node-version: '20'

        - name: Install from npm
          run: |
            VERSION=${GITHUB_REF_NAME#npm-v}
            npm install -g goodvibes-cli@$VERSION

        - name: Smoke test
          run: |
            TMPDIR=$(mktemp -d)
            cd $TMPDIR
            goodvibes init --dry-run 2>&1 | tee /tmp/gv-out.txt
            grep -q "CLAUDE.md" /tmp/gv-out.txt

    Notes for npm smoke-test:
    - `GITHUB_REF_NAME` is e.g. `npm-v1.7.0`; the `#npm-v` strip produces `1.7.0`.
    - `npm install -g` requires Node setup — include `actions/setup-node@v6` step with node-version: '20'.
    - No checkout step needed — smoke test installs from registry, not from source.
    - `needs: publish` means the job only runs if `publish` succeeds.
    - The smoke test pipes output to tee so both stdout and the grep assertion work correctly. `--dry-run` prints file names to stdout but does not write files to disk, so assert on output text (grep for "CLAUDE.md"), not file presence.

    In .github/workflows/publish-pip.yml, append a new job `smoke-test` after the existing `publish` job:

    smoke-test:
      needs: publish
      runs-on: ubuntu-latest
      steps:

          uses: astral-sh/setup-uv@v7

        - name: Wait for registry propagation
          run: sleep 30

        - name: Install from PyPI
          run: uv tool install goodvibes-cli

        - name: Smoke test
          run: |
            TMPDIR=$(mktemp -d)
            cd $TMPDIR
            goodvibes init --dry-run 2>&1 | tee /tmp/gv-out.txt
            grep -q "CLAUDE.md" /tmp/gv-out.txt

    Notes for PyPI smoke-test:
    - No version pinning needed for PyPI smoke-test — installs latest, which was just published.
    - `uv tool install goodvibes-cli` puts the `goodvibes` binary on PATH automatically.
    - `needs: publish` ensures this only runs after a successful publish.
    - No checkout step needed.
    - Same output assertion pattern as npm: pipe to tee, grep for "CLAUDE.md" in output.

    Do not change any existing steps in either publish workflow.
