  Create .github/workflows/publish-template.yml — a manually triggered workflow that pushes the contents of templates/ to jgiox/goodvibes-template.

  The workflow:
    - Triggers on workflow_dispatch only (manual trigger — no automatic push per RESEARCH.md Q10 and Q3)
    - Runs on ubuntu-latest
    - Checks out the monorepo (with full history for the push step)
    - Pushes ONLY the contents of templates/ (not the templates/ directory itself) to jgiox/goodvibes-template main branch
    - Uses git subtree push or a manual git worktree approach — whichever avoids shell:true and shell injection

  Recommended implementation (Ponytail: fewest steps, no new tools):
    Step 1: actions/checkout@v4 with fetch-depth: 0
    Step 2: Configure git user (name: github-actions[bot], email: github-actions[bot]@users.noreply.github.com)
    Step 3: Use git subtree push:
      git subtree push --prefix=templates https://x-access-token:${{ secrets.TEMPLATE_REPO_TOKEN }}@github.com/jgiox/goodvibes-template.git main
    This command pushes only the templates/ subtree as the root of goodvibes-template.

  Secrets required:
    - TEMPLATE_REPO_TOKEN: a GitHub Personal Access Token with repo scope for jgiox/goodvibes-template
    - Document in the workflow as an env comment: "Create at: github.com/settings/tokens — classic PAT, repo scope; add as repo secret at Settings → Secrets → Actions"

  After creating publish-template.yml, perform the INITIAL push manually:
    1. Check that git subtree is available: `git subtree list` or `git subtree --help` (built into git >=1.7)
    2. If git subtree is available: run the initial push from the local dev machine:
       git subtree push --prefix=templates https://github.com/jgiox/goodvibes-template.git main
       (This requires the jgiox/goodvibes-template repo to already exist — created by the user in user_setup above)
    3. If git subtree push fails due to history: alternative is to clone into a temp dir, copy templates/ contents, commit, push:
       - mkdir /tmp/template-push && cd /tmp/template-push
       - git init && git remote add origin https://github.com/jgiox/goodvibes-template.git
       - cp -r /path/to/goodvibes/templates/. .
       - git add . && git commit -m "chore: sync templates from goodvibes v1.0.0"
       - git push -u origin main --force (first push to empty repo is safe)

  Note: The user must have created jgiox/goodvibes-template (empty, public) BEFORE this step runs. The human-verify checkpoint (Task 2) confirms the result. If the push fails here, the checkpoint will catch it.

  The workflow YAML must:
    - Not hardcode any secrets in the file (use ${{ secrets.TEMPLATE_REPO_TOKEN }})
    - Not use shell: true
    - Not push monorepo cruft (.planning/, packages/, .github/workflows/publish-*.yml)

  Verify the workflow is syntactically valid YAML by reading it back after write.
