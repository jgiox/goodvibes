    REQUIRED STRUCTURE AND CONTENT (per D-12):

    Title: `# Getting started`

    Opening paragraph (2-3 sentences): Explain that git is a tool for tracking changes to
    code, that GitHub is the website where code is shared and collaborated on, and that
    this guide walks through the essential steps to make your first contribution. Keep it
    reassuring — no jargon.

    Section: `## Clone the project`
    Explain that "cloning" means downloading a copy of the project to your computer.
    Provide the command: `git clone <repository-url>`
    Explain where to find the repository URL (green "Code" button on GitHub, copy the HTTPS URL).
    After cloning: `cd <project-folder-name>` to enter the directory.
    Explain that you now have a complete local copy including all history.

    Section: `## Create a branch`
    Explain that a branch is a separate workspace for your change, so your work does not
    affect the main codebase until it is reviewed. This is the standard way open source works.
    Commands:
    ```
    git checkout -b my-feature
    ```
    Explain: give your branch a short, descriptive name (e.g., `fix-typo`, `add-dark-mode`).
    Mention: `git branch` shows all branches; the asterisk marks the current one.

    Section: `## Make changes and commit`
    Explain what a commit is: a saved snapshot of your changes with a message describing what changed.
    Steps:
    1. Edit the files you want to change and save them.
    2. Stage your changes: `git add -A` (stages all changed files) or `git add path/to/file` (stages one file).
    3. Commit: `git commit -m "Describe what you changed"`
    Advice on commit messages: one sentence, present tense, describing what the commit does
    (e.g., "Fix typo in README" not "fixed stuff").
    Explain: you can make multiple commits on one branch — each is its own checkpoint.

    Section: `## Push your branch`
    Explain that pushing sends your local branch to GitHub so others can see it.
    Command: `git push origin my-feature`
    (Replace `my-feature` with your actual branch name.)

    Section: `## Open a pull request`
    Explain that a pull request (PR) is a request to merge your branch into the main codebase.
    It lets maintainers review your work before it is merged.
    Steps:
    1. Go to the repository on GitHub.
    2. GitHub usually shows a banner: "You recently pushed a branch — open a pull request." Click the button.
    3. If the banner is not there: click "Pull requests" tab → "New pull request" → select your branch.
    4. Write a clear title and description. Explain what your PR does and why.
    5. Click "Create pull request".
    A maintainer will review and either approve, request changes, or ask questions.

    Closing line: "That is the full loop: clone → branch → commit → push → pull request.
    Every contribution, large or small, follows this same pattern."
